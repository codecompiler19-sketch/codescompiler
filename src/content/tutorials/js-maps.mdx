A `Map` structure holds complex multidimensional data. Unlike a normal JavaScript flat object mapping text strings to standard numbers, a `Map` permits mapping ANY format of specific data to another format natively.

### Constructor Frameworks
You generate the structure deploying a `new Map()` instance node. To inject coordinates, deploy `.set(key, value)`. 

<Editor 
    initialHtml={`<h3>Station Reactor Outputs</h3>\n<p id="reactor-logs"></p>`}
    initialJs={`// Boot up local map node\nconst sectorPowerLevels = new Map();\n\n// Assign power outputs per sector module\nsectorPowerLevels.set("Oxygen Reclaimers", 94);\nsectorPowerLevels.set("Shield Generators", 100);\nsectorPowerLevels.set("Thruster Bays", 42);\n\ndocument.getElementById("reactor-logs").innerHTML = \n   "Current shield array capacitor percentage is at: " + sectorPowerLevels.get("Shield Generators") + "%";`}
/>

### Retrieving Information and Volumes
By executing `.size`, the JS engine instantly assesses the full volume capacity of the current Map. Furthermore, mapping structures execute flawless iterable `forEach` blocks natively!

<Editor 
    initialHtml={`<h3>Systems Diagnostic Map</h3>\n<p id="diagnostic-hub" style="font-family: monospace;"></p>`}
    initialJs={`const systemIntegrity = new Map([\n  ["Comms", "ONLINE"],\n  ["Warp Core", "OFFLINE"],\n  ["Lifesupport", "ONLINE"]\n]);\n\nlet screenReadout = "Total modules mapped: " + systemIntegrity.size + "<br><br>";\n\nsystemIntegrity.forEach (function(status, moduleName) {\n  screenReadout += moduleName + " diagnostic states currently: " + status + "<br>";\n})\n\ndocument.getElementById("diagnostic-hub").innerHTML = screenReadout;`}
